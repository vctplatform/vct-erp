# /sprint-cycle — Sprint Management Workflow

## SPRINT PLANNING (Start of sprint)
// turbo
1. Review backlog priorities
2. Select items for sprint (capacity-based)
3. Break stories into tasks
4. Estimate effort
5. Define sprint goal

## DAILY CHECK (Each day)
```
3 questions per task:
1. What was completed yesterday?
2. What's planned today?
3. Any blockers?
```

## SPRINT REVIEW (End of sprint)
// turbo
1. Demo completed features
2. Review metrics:
   - Velocity (story points completed)
   - PRs merged
   - Test coverage
   - Deployment count
3. Gather feedback

## RETROSPECTIVE
// turbo
1. What went well? (Keep doing)
2. What didn't go well? (Stop/Change)
3. Action items for next sprint (max 2)
